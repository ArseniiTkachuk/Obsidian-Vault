#### JSON (JavaScript Object Notation) 

^1c7535

Це формат обміну інформацією між компютерами. 
Він дуже зручний адже створений таким чином щоб його могли читати й писати не тільки компютери а й люди.
Використовується для описання якихось властивостей та обєктів 
Часто використовують в технології [[Rest API]] для обміну інформацією між [[Frontend]] та [[Backend]].
Також часто використовується в [[No sql]] базах даних для зберігання інформації, як от наприклад [[MongoDB]].

### Синтаксис JSON
Файли JSON мають наступний синтаксис:
оголошується назва, після через двокрапку пишеться значення (масив, рядок, число, булеве значення то що), на кінці кожного елементу кладеться кома, Також після двокрапки можна використати фігурні дужки що означатиме що ми створили новий обєкт в середині іншого обєкту. При відкритті пустого файла json обєкт в якому ми знаходимося є файлом тобто його імя це імя файлу, само тому при відкритті пустого json файлу ми відкриваємо фігурні дужки і працюємо в них.


#### Приклад json файлу
```json
{
   "firstName": "Іван",
   "lastName": "Коваленко",
   "address": {
       "streetAddress": "вул. Грушевського 14, кв.101",
       "city": "Київ",
       "postalCode": 21000
   },
   "phoneNumbers": [
       "044 123-1234",
       "050 123-4567"
   ]
}
```

 Тут є [[Змінна]] "firstName" яка має значення "Іван", аналогічно з другим рядком, а от уже в третьому оголошується [[Змінна]] "assress" яка є обєктом у якому два перших значення це рядки а останнє це число, а от четвертий елемент нашого прикладу це [[Масив]] 

#### Звернення до значення
Для звернення до якогось значення пишеться шлях по обєкту json через крапку, наприклад треба дістати значення зі що є в змінній "city"
пишемо: назва файлу.address.city
